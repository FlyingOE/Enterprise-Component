parseq.q

Parseq library

Parseq is a Q clone of the Haskell’s Parsec, a parser combinator library.  Most parser names are the same as in the original, except when colliding with q keywords.

Notes

All parsers described here take a dictionary describing the parse state with key `s`cp`errp`errs`ast as the last parameter.

The meaning of these fields is as follows :

-- sthe string under parse.  This is not modified by the parser
-- cpthe cursor position: the index of the next character in s to be parsed
-- errperror position.  This is integer null (0N) if there is no error.
-- errsa list of error strings
-- astthe Abstract Syntax Tree: the value returned from parser

For brevity the parse state parameter is not included in function documentation.

All parsers return the parse state dictionary.

The return value described in the documentation concerns the `ast field of the parse state dictionary.

chainl[pa;op;x0;ps]

Parses zero or more occurrences of a pattern, separated by another one.  Always succeeds.

Parameter

paparser for the pattern
opa parser for the separator.  This parser should return dyadic (binary) function (in the ast field).
x0default value to be returned in case no pa is matched

Returns

Returns:a value obtained by a left associative application of all functions returned by op to the values returned by pa.  If there are zero occurrences of pa, the value x0 is returned.

chainr[pa;op;x0;ps]

Applies a list of operations to a list of arguments (this is not a parser)

Parameter

opsa list of dyadic (binary) operations - one shorter than the list of arguments.  This condition is not checked.
argsa list of arguments

Returns

the result of the right-associative application of the operation to the list of arguments.

Example

applyChainr[(f0;f1;f2);(x0;x1;x2;x3)] = x0 f0 (x1 f1 (x2 f2 x3)) (writing f's infix)
applyChainr[({x-y};{x*y};{x+y};(3;2;1;0)] = 1

Returns

Parses zero or more occurrences of a pattern, separated by another one.  This parser always succeeds.

Returns

a value obtained by a right associative application of all functions returned by op to the values returned by pa.  Returns x0 if there are no occurrences of the pattern.
