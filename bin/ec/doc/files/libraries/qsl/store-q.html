store.q

Store library

Library for data storage in the hdb (hdb.q)

Provides

  • data storage as splayed and partition table
  • callback for reloading and filling missing tables in hdb (hdb.q)
  • end of day support and integration with eodMng (eodMng.q) component

.store.init[config;reloadHdb;fillMissingTabsHdb;dataPath]

initialization of store library

Parameter

config:TABLEthat contains:
-- table:SYMBOLtable name
-- hdbPath:SYMBOLpath to the hdb; path to hdb location is taken automatically from system.cfg (field: dataPath) for given hdb name
-- hdbName:SYMBOLconnection to reload hdb
-- memoryClear:BOOLEANflag for clearing data after data storing is performed
-- store:BOOLEANflag for performing data storing
-- attrCol:SYMBOLcolumn which should have attribute p applied (optional column in the configuration table, if missing will be filled with `sym)
reloadHdb:BOOLEANflag for reloading hdb after storing
fillMissingTabsHdb:BOOLEANflag for filling missing tables in hdb after storing is completed
dataPath:PATHpath to the directory that can be used for storing status of the store procedure

Example

.store.init[flip (`table`hdbPath`hdbName`memoryClear`store!(enlist `quote;enlist `:hdb;enlist `kdb.hdb;enlist 1b; enlist 1b));1b;1b;`:data/]

.store.run[date]

Perform storing procedure; main top level function

  • data storing and cleaning
  • invoke callbacks for reloading and filling missing tables in partitions
  • .store.run should be proceeded by the <.store.init> with proper data storing settings

Parameter

date:DATEeod date (in most cases current date)
