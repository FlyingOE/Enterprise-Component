Variables

stopWords (LIST STRING)

Stop words definition.  Definition should be put at the top-level of access.cfg file.

Example

stopWords = .z.exit,.z.po,.z.pc,.ap.cfg.serverAux,set,upsert

auditView (LIST SYMBOL)

Enables auditing view.  Auditing view is used to log additional information on open/closed connections and function calls (both synchronous or asynchronous).  It can be set on global level and later overridden on userGroup or component level.  Possible values are listed below :

NULLaudit is turned off
CONNECTIONS_INFOview to connections is turned on
SYNC_ACCESS_INFOview to synchronous messages is turned on
ASYNC_ACCESS_INFOview to asynchronous messages is turned on
CONNECTIONS_INFO, SYNC_ACCESS_INFO,ASYNC_ACCESS_INFOturn on all views

Example

auditView = CONNECTIONS_INFO, SYNC_ACCESS_INFO

checkLevel (SYMBOL)

Check level for user query (as in authorization.q)

NONEno check is performed (default)
STRICTcheck function name for allowed namespaces; check one level of nested arguments (allow tables, lists, dictionaries but block inline function calls); only parse trees arguments with function names given as either symbol or string
FLEXcheck function name for allowed namespaces; check all function parameters against stopWords (see access.qsd); allow parse trees (as in STRICT mode) or string calls

Example

(`function; arg1; arg2)    / allowed on both STRICT and FLEX levels
("function"; arg1; arg2)   / allowed on both STRICT and FLEX levels
"function[param1;param2]"  / allowed on FLEX level

[technicalUser:<type(STRING)>] (STRING)

Technical user specification.  Technical user is used for inter-process communication inside Enterprise Components.  At least one technical user has to be configured in the system

Example

[technicalUser:superadmin]
  pass = 0xS00P3rP455
  usergroups = admin

pass (STRING)

Encrypted password

usergroups (LIST SYMBOL)

Technical user’s group assignments.

[userGroup:<type(SYMBOL)>] (SYMBOL)

User group permissions

Example

[userGroup:admins]    # example group with full access
  [[ALL]]             # all namespaces on all components, no checks
  namespaces = ALL
  checkLevel = NONE
  [[rtr.feed]]        # restrict access to rtr.feed
  namespaces = .admin # only .admin namespace is allowed
  checkLevel = STRICT # allow only interface calls in parse tree form
