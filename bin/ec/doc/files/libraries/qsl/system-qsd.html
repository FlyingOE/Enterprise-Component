timestampMode (SYMBOL)

Run system in UTC or local (LOCAL) time.  This will affect messages’ timestamping, log messages’ timestamping and log rotating, timer callbacks and end-of-day broadcasting.  To enable KDB_TIMESTAMP_MODE in the custom code there is need to use interface functions described in sl.q

eodDelay (TIME)

End-of-day delay.  If greater than 00:00:00.000 will delay end-of-day procedure.  E.g. will enable late ticks handling and delay eod broadcasting in tickHF(tickHF.q), delay end-of-day procedure on tickLF (tickLF.q), rdb (rdb.q), stream (stream.q), eodMng (eodMng.q) and hdb queries that are using interface functions from query.q.  To get the current date that includes EC_EOD_DELAY there is need to use .sl.eodSyncedDate[] function described in sl.q

Wait period for process startup

stopWait (FLOAT)

Wait period for process stop

libPath (LIST PATH)

qsl library path.  List of directories searched for requested q modules

dllPath (LIST PATH)

qsl shared object path.  List of directories searched for shared objects

etcPath (PATH)

qsl configuration path.  Directory searched for configuration files

logLevel (SYMBOL)

Logging level.  Sets the log level (one of DEBUG, INFO, WARN, ERROR or FATAL)

logDest (LIST SYMBOL)

Destination (or collection of destinations) where the log is written.  Currently only FILE, CONSOLE and STDERR are supported.

CONSOLEwrite log messages to the console
FILEwrite messages to a log file in $logPath.
STDERRwrite ERROR and FATAL level messages to the standard error output and ignores messages on other levels.

Destinations are separated by commas in the environment variable

logRotate (TIME)

Time for log rotate (specified according to KDB_TIMESTAMP_MODE)

cpuAffinity (LIST INT)

CPU affinity settings.  Can be overridden on group or component level.  Multiple CPU number are allowed.

User-access option to use.  See u/U command line options on http://code.kx.com

User-access file to use

eventPath (PATH)

Location to store event files.  (see event.q library)

export (LIST SYMBOL)

List of variable patterns to export.  Configuration variables matching patterns specified here are automatically added to the environment created for component execution.

housekeeping (ARRAY)

Actions to perform during housekeeping.  Housekeeping is described in hk.q.

subAutoSubscribe (BOOLEAN)

If set to TRUE and qsl/sub_dist.q library loaded, subscription to dist server will be done automatically, otherwise <.sub.init[]> must be invoked in the custom code

commonLibs (LIST COMPONENT_TYPE)

List of common libraries that should be loaded to all processes in system.cfg file or within one [group] This command option is analogical to the field `libs`.  Note that it is applied before `libs` field.

[group:<type(SYMBOL)>] (SYMBOL)

Group definition.  Grouping allows creating collections of components for easier reference.  Groups can be passed as arguments to `yak’ start/stop/info commands.

Example

[group:core]
  [[rtr.feed]]
  # component instance parameters go here
  [[rtr.tick]]
  # ...
  [[rtr.rdb]]
  # ...
  [[rtr.hdb]]
  # ...

[[<type(SYMBOL), isComponent()>]] (SYMBOL)

Component instance definition.  Name of the instance should consist of ‘namespace’ prefix.  Namespaces can be used to group similar functionalities (for instance input, batches, admin)

Example

[[rtr.hdb]]                       # give a name
  command = "q"                   # run command, chdir to $binPath first
  type = q:hdb/hdb                # this is a q-based process
  requires = rtr.rdb              # start only if rtr.rdb is running
  port = ${basePort} + 10         # simple arithmetic operations are allowed
  multithreaded = TRUE            # give hdb negative port number = multithreaded mode

type (COMPONENT_TYPE)

Type of instance.  Mandatory value describing type of the component.  Type consists of colon-separated typeid and description (description part is not used).

c:external connection.  Definition of a component deployed on some other site.  Used for connections only.
q:q-based process definition.
cmd:external command.  Used when integrating external tools into the system.

libs (LIST COMPONENT_TYPE)

List of additional libraries

requires (LIST SYMBOL)

List of required component dependencies.  If not empty, listed components have to run before starting this instance.

port (PORT)

Port number (type q: instances only).  Basic arithmetic operations are allowed.

command (STRING)

Command to execute.  Command starting defined instance (valid only for q: and cmd: types)

host (STRING)

Host name (type c: instances only).  Valid only for external connections connections (type=c)

multithreaded (BOOLEAN)

Multithreaded execution mode (type q: instances only).  Run configured instance with negative port as described on http://code.kx.com/wiki/Cookbook/MultithreadedInputMode
