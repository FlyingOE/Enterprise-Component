.sl.setConfVar[vn;envn;default;conv]

Sets a configuration variable.  This does not have to be called for all configuration variables, just for those for which we may have corresponding environment variable and default value.  The algorithm works as follows:

  • if the variable is already defined, do nothing, it must have been set by <.sl.load[`ETC]>
  • if not, check for the environment variable and set to its parsed (by the provided parser) value
  • if the variable does not exist, use the default

Parameter

vn:SYMBOLthe q configuration variable name
envn:SYMBOLcorresponding environment variable name
default:ANYthe default value to be used
conv:FUNTIONa function that converts the string from environment variable to the value of configuration variable.

Returns

ANY - actually assigned configuration variables

.sl.getConfStatus[]

Function that shows the values of environment configuration variables.  The `name column shows the name of the (shell) configuration variable and `source shows if the value was taken from a q variable (usually set in a configuration file), environment or was taken as default in the absence of both.

.sl.getSubProtocols[]

Information about subscription protocols supported by the component (if the are any).  This function should be overwritten by the component code in order to return proper values.  This function is used by qsl/sub library to choose proper subscription protocol.

Returns

SYMBOL: returns list of protocol names that are supported by the server:

  • `PROTOCOL_TICKHF - see tickHF, stream and monitor components
  • `PROTOCOL_TICKLF - see tickLF component
  • `PROTOCOL_DIST - see dist component
  • or other custom protocol SYMBOL: returns `symbol$() if it is not possible to subscribe to the server.
