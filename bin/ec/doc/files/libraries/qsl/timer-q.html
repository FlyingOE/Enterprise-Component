Functions

.tmr.start[func;per;id]

runs a function every number of timer ticks provided per parameter

Parameter

func:SYMBOLname of a function that takes exactly one parameter; the timestamp from .z.p or .z.P (depending on the EC_TIMESTAMP_MODE configuration variable) is passed to that function
per:INTthe period between runs in ms
id:SYMBOLtimer id, assigned by the user; if such already exists, the previous is overwritten

Returns

INT - number of timers defined (including this one)

.tmr.stop[id]

removes a function (timer callback) from the list to be run; other callbacks are not affected.

Parameter

id:SYMBOLid of the timer to be removed

Returns

INT - number of remaining timers

.tmr.change[per;id]

changes the frequency of given timer

Parameter

per:INTnew timer period in ms
id:SYMBOLtimer identifier

Returns

Intprevious period in ms

.tmr.reset[]

removes all timer callbacks

.tmr.runAt[t;func;id]

runs the specified function around specified time (10 s accuracy)

Parameter

t:TIMEthe time the function is to run the first time
fun:SYMBOLthe name of the function
id:SYMBOLthe id of this job

.tmr.sanityCheck[]

checks for known invariants in case someone resets the q timer or .z.ts by hand; errors are logged, but no other action is taken (no signals)

Returns

message about the state of timer

.tmr.run[]

runs a timer function on the timer tick; if some tick have been missed because the process was busy, each missed function is executed once
