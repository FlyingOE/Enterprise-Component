[table:universe]
   dirSrc =  ${dataPath}/universe
   pattern = *universe*
   separator = ;

   Configuration examples:

   [table:universe]
   model = time(TIME), sym(SYMBOL), underSym(SYMBOL)

   1. Configuration example if file is matching the data model completely (header and column types are the same as in data model).
      Fields fileModel and file2Tab are taken as defaults.

      Csv file example:

         time;sym;underSym
         00:00:00.000;instrument;underlying

      Configuration example:

         [[kdb.feedCsv]]
         dirSrc =  ${dataPath}/universe
         pattern = *universe*
         separator = ;

   2. Configuration example if file is matching the data model completely, but csv file doesn't contain header.
      Fields fileModel and file2Tab are taken as defaults.

      Csv file example:

         00:00:00.000;instrument;underlying

      Configuration example:

         [[kdb.feedCsv]]
         dirSrc =  ${dataPath}/universe
         pattern = *universe*
         separator = ;
         headerInFile = FALSE

   3. Configuration example if file is not matching the data model and custom plugin is required.

      Csv file contains tmp column that should be split into sym and underSym columns:

         time;tmp
         00:00:00.000;`instrument`underlying

      In this case there is need to configure fileModel field and plugin .fcsv.plug.enrich[`universe]:

         [[kdb.feedCsv]]
         dirSrc =  ${dataPath}/universe
         pattern = *universe*
         separator = ;
         fileModel = time(TIME), tmp(STRING)

   4. Configuration example if file is not matching the data model and reorder of columns from csv file is required.

      Csv file example:

         time;underSym;sym
         00:00:00.000;underlying;instrument

      In this case there is need to configure fileModel and file2Tab fields:

         [[kdb.feedCsv]]
         dirSrc =  ${dataPath}/universe
         pattern = *universe*
         separator = ;
         fileModel = time(TIME), underlying(SYMBOL), sym(SYMBOL)
         file2Tab = time(time), sym(sym), underSym(underlying)

   5. Configuration example if file is not matching the data model and there is need to parse selected columns and assign proper order.

      Csv file example without header:

         00:00:00.000;notRequired;underlying;instrument

      Configuration example with using artificial column names col1, col2, etc.:

         [[kdb.feedCsv]]
         dirSrc =  ${dataPath}/universe
         pattern = *universe*
         separator = ;
         headerInFile = FALSE
         fileModel = col1(TIME), col3(SYMBOL), col4(SYMBOL)    # parse only selected columns col1, col3, col4 from csv file
         file2Tab = time(col1),sym(col4),underSym(col3)        # add order here that is matching the data model

   6. Configuration example if file is not matching the data model and there is need to parse selected columns.

      Csv file example without header:

         00:00:00.000;instrument;underlying;notRequired

      Configuration example with using artificial column names col1, col2, etc.:

         [[kdb.feedCsv]]
         dirSrc =  ${dataPath}/universe
         pattern = *universe*
         separator = ;
         headerInFile = FALSE
         fileModel = col1(TIME), col2(SYMBOL), col3(SYMBOL)    # parse only selected columns col1, col2, col3 from csv file
         file2Tab = time(col1),sym(col2),underSym(col3)        # add order here, select only required columns

[table]

dirSrc (PATH)

absolute path to a directory that has to be watched

Example

dirSrc =  ${dataPath}/universe

pattern (STRING)

pattern to match against files found in directory specified above; note that parsed files that are pushed to tickLF should include in the file names: upd or img (e.g.  [date].upd.[table].csv, [date].img.[table].csv)

Example

pattern = *universe*

fileFormat (STRING)

parse string, deprecated field; please use fileModel and file2Tab instead

Example

fileFormat = TSS

separator (STRING)

column separator

Example

separator = ;

headerInFile (BOOLEAN)

flag to indicate if csv file contains header

Example

headerInFile = TRUE

fileModel (TABLE)

format definition of file as list of pairs: column name and format; if default is used, then order is taken from csv file as it is

Example

fileModel = time(TIME), underlying(SYMBOL), sym(SYMBOL)  # file with header
fileModel = col1(TIME), col2(SYMBOL), col3(SYMBOL)       # file without header

file2Tab (TABLE)

mapping for columns in data model and csv file as list of pairs: column name from data model and column from csv file, if default is used, then order is taken from csv file as it is

Example

file2Tab = time(time), sym(sym), underSym(underlying)    # file with header
file2Tab = time(col1),sym(col2),underSym(col3)           # file without header
