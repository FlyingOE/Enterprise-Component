streamMrvs.q

Most recent values (mrvs) stream plugin component

Responsible for

  • capturing and storing most recent values

Notes

  • mrvs is subscribing for selected tables to the tickHF server
  • data model of each subscribed table is modified, all subscribed tables are keyed by sym column and suffix “Mrvs” added to table names
  • mrvs tables keep exactly one (most recent) record per symbol that came for each table
  • update frequency is throttled by <.plug.ts> providing lower cpu usage
  • mrvs state can be dumped every N minutes
  • mrvs tables are updated on ts callback
  • data is cached in original table names and deleted on ts callback
