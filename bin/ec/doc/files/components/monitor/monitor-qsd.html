[group]

cfg.procMaskList (LIST SYMBOL)

List of monitored processes is based on .monitor.cfg.procMaskList process masks list.  Mask list is applied to process names defined in the main system configuration file.  Use `ALL option in order to monitor all processes from the system.

Example

cfg.procMaskList = kdb, admin

cfg.jrn (PATH)

Journal location

Example

cfg.jrn = ${dataPath}/monitor

cfg.eventDir (PATH)

Directory with event files

Example

cfg.eventDir = ${eventPath}

cfg.monitorStatusPublishing (BOOLEAN)

Detailed monitor status publishing (table sysMonitorStatus), generates a lot of load, switched off by default.

Main status information table describing overview and general state of all monitored components.

sysStatus data model.

[sysTable:sysMonitorStatus]

Table describing actual monitoring status published to the subscribers.  One row contains information about state of check on each server.  This table generates a lot of traffic, should be used only when profiling the system.  By default turned off, can be turned on by .monitor.cfg.monitorStatusPublishing configuration option.

frequency (INT)

Frequency of updates, specified in milliseconds

model (TABLE)

sysMonitorStatus data model.

-- time:TIMEtimestamp of published information
-- sym:SYMBOLcomponent name
-- hndStatus:SYMBOLcomponent handle status
-- request:SYMBOLrequest id
-- check:SYMBOLrequest name
-- code:STRINGrequest code
-- ts0_req:TIMESTAMPtimestamp of sent request
-- ts1_befExec:TIMESTAMPbeginning of request execution
-- ts2_afterExec:TIMESTAMPend of request execution
-- ts3_res:TIMESTAMPtimestamp of result retrieving
-- status:SYMBOLstatus of the check
-- interval:INTinterval of check execution
-- requestId:LONGcurrent request id
